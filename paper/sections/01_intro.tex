\section{Introduction}
Steganography aims to hide the \emph{existence} of communication, not only its plaintext content. In text channels, this objective is difficult because language is discrete and highly structured: small distributional distortions can expose artifacts to a detector, while strict decoding dependence on generation details can make communication brittle. Classical and provable-security formulations established the conceptual foundations~\cite{DBLP:conf/crypto/Simmons83,DBLP:journals/jsac/AndersonP98,DBLP:conf/crypto/HopperLA02}; recent neural and LLM-based approaches show that arithmetic/interval coding over language-model distributions can embed payload bits in plausible text~\cite{DBLP:conf/acl/FangJA17,ziegler-etal-2019-neural,DBLP:conf/acl/ZhangYYH21,DBLP:conf/naacl/LinLZL24}.

However, practical covert messaging systems must satisfy a property that is often under-specified in generation-centric work: \emph{operationally reliable recovery under deployment constraints}. A receiver should recover payloads when sender and receiver are synchronized, and the system should fail closed when synchronization assumptions are violated. In modern LLM pipelines, subtle mismatches---prompt template drift, tokenizer behavior, backend settings, or runtime configuration differences---can silently invalidate decode paths. These failure modes are security-relevant: a system that appears to decode but returns incorrect plaintext is dangerous for covert communication.

This paper studies \textbf{Ghostext}, a security-oriented engineering system for local-LLM text steganography. Ghostext does \emph{not} claim broad undetectability, active-channel robustness, or cross-model interoperability. Instead, it targets a narrower but actionable goal: deterministic recoverability under explicit synchronization assumptions, and explicit fail-closed behavior otherwise.

Ghostext combines: (i) encrypt-then-steganography packetization, (ii) candidate-set constraints with tokenization-path safety checks, (iii) integer quantization and finite-interval decoding, and (iv) configuration fingerprint validation that binds runtime assumptions to the packet-level decryption path. This design philosophy aligns with security-systems practice: constrain assumptions, surface boundaries, and make failures explicit.

Our contributions are:
\begin{itemize}
\item A threat-model-explicit protocol and implementation for local LLM text steganography that prioritizes deterministic recoverability and fail-closed decoding.
\item A practical synchronization discipline that integrates packet cryptography, candidate policy control, retokenization-stability filtering, and configuration-fingerprint checks.
\item A reproducible evaluation focused on three security-relevant axes: matched-configuration correctness, fail-closed mismatch behavior, and practical communication overhead.
\end{itemize}
