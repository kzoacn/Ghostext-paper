\section{Introduction}

Steganography and cryptography solve different problems. Cryptography protects message content, while steganography aims to hide that communication is taking place at all. In text channels, both lexical quality and token-level statistics can reveal generation artifacts.

Neural Linguistic Steganography (NLS)~\cite{ziegler2019neural} established arithmetic-coding-based LLM steganography as a practical baseline, and OD-Stega~\cite{huang2026odstega} extended this line with constrained-distribution optimization and explicit tokenization-mismatch discussion. Ghostext follows the same generation-based family but emphasizes a different systems-security question: what exact bitstream is handed to the arithmetic-coding cover generator, and how robustly can that bitstream be replayed in an LLM backend? Ghostext answers it with a passphrase-bound AEAD packet, deterministic synchronization rules, and retokenization-stability filtering. We position the design as \emph{near-distribution-preserving}: against the same-policy natural-cover baseline, top-$p$/max-$k$ truncation is shared and cancels, leaving only integer quantization (bounded by $1/F_{\mathrm{tot}}$ per candidate) and the retokenization-stability filter as residual per-step deviations. Full perfect-security claims in the Cachin / Hopper / de Witt sense are out of scope for this paper.

\paragraph{Contributions.} (i) An implementation-backed protocol description with authenticated packet framing, cryptographic payload normalization, and runtime configuration binding. (ii) Deterministic candidate selection with integer quantization and retokenization-stability filtering. (iii) LLM-backend evidence on Qwen/llama.cpp: an 18-trial bilingual template-known baseline for recoverability/runtime/efficiency and a six-case same-backend slice against implementation-aligned NLS-style single-segment baselines (Section~\ref{sec:aligned-slice}).

\paragraph{Scope.} The 18 trials are a pilot measurement, not a settled reliability estimate. Detector-facing evaluation, active text modification in transit, and cross-hardware determinism are out of scope in this version.
