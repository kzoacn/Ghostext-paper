\section{Evaluation}

We keep the current evaluation narrow and tied to the released implementation. The CLI already exposes a minimal end-to-end benchmark through \texttt{ghostext benchmark}, so this section reports that benchmark directly instead of adding extra aggregate metrics beyond the shipped interface. The benchmark runs a full encode/decode round trip and returns average encode latency, average decode latency, average encode bits/token, and average perplexity over repeated runs.

\subsection{Simple End-to-End Benchmark}

Table~\ref{tab:simple-benchmark} reports the structured JSON output of running \texttt{ghostext benchmark} with the local \texttt{llama-cpp} backend, the README quiet-evening-walk prompt, passphrase \texttt{demo-pass}, message \texttt{Attack at Dawn!}, \texttt{--runs 3}, and \texttt{--json}. Because the benchmark aborts if any round trip fails, successful completion already implies correct recovery for all three runs; we therefore do not add a separate success-rate row. All other CLI settings stay at their released defaults, so the table is best read as a reproducible snapshot of the default local path rather than a tuned best-case configuration.

Two points matter when interpreting Table~\ref{tab:simple-benchmark}. First, the reported latencies are arithmetic means across three complete runs, so they describe the current default path but do not characterize tail behavior. Second, the perplexity value is computed against the same backend distribution over the generated continuation and should be read only as a rough fluency check. It is not a detector result and does not by itself establish indistinguishability from natural text.

Within that limited scope, the benchmark supports one modest claim: the released Ghostext CLI completes stable end-to-end round trips on the default local \texttt{llama-cpp} path while exposing transparent runtime and payload-density numbers through a single reproducible command. Broader claims about detector resistance, robustness under editing, or comparison against prior systems remain future work unless measured directly in this project.
