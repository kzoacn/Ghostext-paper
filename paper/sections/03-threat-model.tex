\section{Threat Model and Claim Discipline}

\subsection{Roles and Trust Assumptions}

Ghostext uses three roles: Alice (sender), Bob (receiver), and Censor (observer). Alice and Bob share a passphrase and protocol-sensitive configuration. The Censor observes transmitted cover text.

The synchronized configuration includes at least the model family and tokenizer behavior, prompt text, seed policy, candidate policy, quantization parameters, and codec/crypto settings. This assumption is strict by design. Ghostext does not attempt approximate decoding under drifted settings.

\subsection{Primary Adversary}

The primary adversary in this draft is a \textbf{passive censor}. The censor can read cover text but does not modify it in transit. The censor does not know the shared passphrase and does not know the exact prompt.

Under this scope, the security objective is distributional plausibility: stego text should not be easily distinguishable from normal model outputs under comparable contexts. The reliability objective is exact recovery by Bob when configurations match.

\subsection{Secondary Adversaries (Not Claimed Yet)}

We explicitly separate stronger adversaries that are not covered by current claims.

An \textbf{active channel adversary} can delete, insert, reorder, or paraphrase text. Ghostext currently provides fail-closed behavior in many mismatch cases but does not claim robust payload recovery under arbitrary edits.

A \textbf{privileged detector} may have side information about prompts, seeds, model versions, or even labeled samples generated by the same encoder. This setting is not yet evaluated in this repository and remains future work.

A \textbf{drift adversary} can exploit model or tokenizer version mismatch between sender and receiver. Ghostext addresses this operationally through configuration fingerprint checks, but does not claim compatibility recovery across drift.

\subsection{Claim Taxonomy}

To keep writing honest during the experiment pause, we classify all statements into three types.

\textbf{Implemented fact}. A statement directly evidenced by current source code and tests in \texttt{../Ghostext}.

\textbf{Inference}. A limited implication consistent with implemented behavior but not yet validated at scale.

\textbf{Future work}. A statement requiring new experiments, broader datasets, or stronger adversary analysis.

This taxonomy is enforced in our notes and appendix mapping.

\subsection{Operational Security Goal in This Draft}

Within this scope, Ghostext pursues a practical security target: if Alice and Bob remain synchronized and the Censor is passive and prompt-unaware, then the communication should be difficult to distinguish from ordinary model output while keeping deterministic message recovery.

We intentionally avoid stronger wording such as ``provably secure'' for the current implementation. The manuscript is designed to remain conservative until detector and robustness evaluations are performed directly in this project.
