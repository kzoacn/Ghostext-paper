\section{Protocol and Implementation Details}
\subsection{Packet Construction and Cryptography}
Ghostext uses an opaque packet v2 format with an encrypted internal header and encrypted body. A master key is derived from passphrase and salt via scrypt, and key separation is then performed via HKDF-SHA256 to obtain independent header and body subkeys. Both header and body use ChaCha20-Poly1305 authenticated encryption. This design means wrong passphrases, tampering, or packet corruption are expected to fail at authenticated decryption rather than return partial plaintext.

At a high level, a packet contains random salt, header nonce with sealed internal header, and body nonce with ciphertext plus authentication tag. The internal header carries protocol-sensitive metadata, including encrypted body length and configuration fingerprint.

\subsection{Candidate Policy and Retokenization Stability}
At each generation step, Ghostext computes the next-token distribution and applies deterministic filtering with top-$p$, max-candidates, and minimum entropy threshold. When retokenization-stability enforcement is enabled, a candidate token is accepted only if appending it, rendering text, and retokenizing yields the same token path. This guard directly targets tokenization drift risks highlighted in recent LLM steganography and watermarking literature~\cite{DBLP:conf/emnlp/YanM25}.

\subsection{Deterministic Quantization and Interval Coding}
Selected probabilities are quantized into integer frequencies that sum to a fixed total frequency. Tie handling follows deterministic token-id ordering to keep encoder and decoder aligned. Ghostext then performs finite-interval coding for both encoding and decoding: the encoder chooses the subinterval containing the payload target, and the decoder narrows the same interval by absorbing observed token indices until the payload segment resolves.

\subsection{Synchronization and Fail-Closed Checks}
Fail-closed behavior is enforced at multiple layers. Ghostext rejects mismatched configuration fingerprints, authenticated decryption failures, unexpected observed tokens outside the expected candidate set, interval-collapse conditions, and token-budget exhaustion. This is intentional: in covert messaging, ambiguous decode states are security-unsafe, so the system does not attempt approximate recovery when synchronization assumptions are violated.
