\section{Open Science}
To evaluate the core claims of this paper, reviewers need four artifact groups: (1) the Ghostext implementation snapshot used for experiments, (2) the test suite that exercises crypto, synchronization, and fail-closed behavior, (3) evaluation scripts in this paper repository for correctness/fail-closed/overhead reporting, and (4) raw JSON outputs from reruns.

For double-blind review, we will provide an anonymous artifact URL and a fixed checksum in the submission version. The artifact bundle will include exact command lines and expected output file names so reviewers can rerun the minimal evaluation pipeline end-to-end. If any component cannot be released in full due to licensing or safety constraints, we will provide a concrete justification and include a partial but sufficient substitute to validate the corresponding claim boundary.
