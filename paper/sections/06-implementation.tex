\section{Implementation Evidence}

Ghostext is implemented in a companion code artifact (anonymized for review). Core modules map directly to protocol stages: packet security (crypto + packet framing), candidate construction and stability checks, integer quantization, interval encode/decode, and end-to-end encode/decode orchestration.

\subsection{Backends}

Current implementation provides a deterministic toy backend and a real llama.cpp Qwen backend. The toy backend is primarily for protocol-level testing and deterministic regression. The real backend is intended for operational use and post-pause experiments.

\subsection{Observed Failure Classes}

The code distinguishes failure classes relevant for later security and reliability analysis: integrity mismatch, configuration mismatch, synchronization mismatch, unstable-tokenization exhaustion, low-entropy retry exhaustion, and token-budget exhaustion.

\subsection{Test Coverage (Implemented Facts)}

Existing tests cover: packet crypto round-trip and mismatch failures; deterministic quantization behavior; interval invertibility in toy settings; wrong-prompt/wrong-seed/text-mutation fail-closed behavior; retokenization stability guard behavior; bilingual toy round trips; and optional real-model smoke tests.

These tests support correctness and fail-closed claims. They do not substitute for full detector-resistance evaluation.
