\section{Implementation}

Ghostext is implemented in a companion code artifact (anonymized for review) whose modules map directly to protocol stages: packet crypto and framing, candidate construction with stability checks, integer quantization, interval codec, and end-to-end encode/decode. We provide a deterministic toy backend for protocol-level regression testing and an LLM backend on \nolinkurl{Qwen/Qwen3.5-2B} via \nolinkurl{llama-cpp-python}. The released configuration pins the third-party GGUF repository \nolinkurl{bartowski/Qwen_Qwen3.5-2B-GGUF}, the file \nolinkurl{Qwen_Qwen3.5-2B-Q4_K_S.gguf}, the upstream base model \nolinkurl{Qwen/Qwen3.5-2B-Base}, and the packaged llama.cpp build \nolinkurl{f49e917}.

The implementation is organized around replay symmetry. On the sender side, Ghostext builds the packet, derives the next-token candidate set from the current prompt and generated prefix, filters unstable candidates, quantizes the remaining mass, and narrows the active interval until the packet is fully embedded. On the receiver side, the same backend and prompt recreate the candidate set step by step from the observed cover text, and the interval updates are inverted until the packet is recovered. No hidden side channel or token annotation is needed: the cover text is the only transmitted object, so any mismatch in replay state becomes part of the protocol semantics rather than an implementation detail.

Released result bundles are designed to make this replay contract auditable. Per-run summaries pin Ghostext and paper git heads, hardware/runtime context, tokenizer hash, backend identifier, \nolinkurl{llama-cpp-python} version, llama.cpp build commit, model provenance, and GGUF metadata. The goal is not only that a run can be repeated, but also that each reported number can be traced back to a concrete software and model stack. This is especially important for local LLM experiments, where small backend or tokenizer changes can otherwise blur whether a failure is algorithmic or environmental.

The test suite is layered accordingly. Unit tests cover packet crypto round-trip and mismatch behavior, deterministic quantization, and interval invertibility in toy settings. Integration tests cover wrong-prompt, wrong-seed, and text-mutation abort behavior, retokenization stability, bilingual toy round trips, and optional \texttt{llama-cpp} smoke tests. These tests support the paper's narrow correctness claims: authenticated packet recovery, deterministic replay under matched state, and explicit failure when state drifts. They do not by themselves establish detector resistance or cross-hardware portability beyond the pinned local environment.
